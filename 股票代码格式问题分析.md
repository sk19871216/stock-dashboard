# 股票代码显示问题分析报告

## 问题现象
在缠论分析页面的全市场扫描功能中，股票代码显示为 "1"、"10" 而不是 "000001"、"000010" 等标准格式。

## 根本原因
数据库中存在**两种不同格式**的股票代码：

1. **标准格式**（6 位数）：5191 个代码
   - 例如：'000001', '000002', '000006' 等
   - 这些是正确的股票代码格式

2. **短格式**（1-2 位数）：8 个代码
   - 例如：'1', '2', '4', '6', '7', '8', '9', '10'
   - 这些可能是之前的测试数据或错误导入的数据
   - 每个都有 1500+ 条历史记录

## 数据统计
- 长度 1 的代码：7 个（10,563 条记录）
- 长度 2 的代码：1 个（1,509 条记录）
- 长度 6 的代码：5191 个（305,389 条记录）

## 受影响的代码位置

### 1. pages/4_缠论分析.py
- **函数**：`get_all_stock_codes_cached()`
- **原代码**：`SELECT DISTINCT code FROM daily ORDER BY code`
- **问题**：返回所有代码，包括短格式代码
- **修复**：`SELECT DISTINCT code FROM daily WHERE LENGTH(code) = 6 ORDER BY code`

### 2. utils/database.py
- **函数**：`get_all_stock_codes()`
- **原代码**：`SELECT DISTINCT code FROM daily ORDER BY code`
- **问题**：返回所有代码，包括短格式代码
- **修复**：`SELECT DISTINCT code FROM daily WHERE LENGTH(code) = 6 ORDER BY code`
- **附加修复**：`SELECT DISTINCT code FROM stock_list WHERE LENGTH(code) = 6 ORDER BY code`

## 修复方案
在 SQL 查询中添加 `WHERE LENGTH(code) = 6` 条件，过滤掉短格式代码，只返回标准 6 位数格式的股票代码。

## 验证结果
修复后：
- ✅ `get_all_stock_codes_cached()` 返回 5191 个 6 位数代码
- ✅ `database.get_all_stock_codes()` 返回 5191 个 6 位数代码
- ✅ 股票代码格式统一为 '000001' 等标准格式

3. **长期监控**：在数据导入时添加格式验证逻辑

## 后续修复（2026-04-04 补充）

### 新发现的问题
检查数据存入数据库的完整流程后，发现以下风险点：

1. **mootdx 返回的数据没有代码列**
   - `mootdx.client.bars()` 返回的 DataFrame 只有 open, close, high, low, vol, amount 等列
   - **没有 code 或 symbol 列**
   - 保存到数据库时完全依赖传入的 code 参数

2. **akshare 返回的数据是标准的**
   - `akshare.stock_info_a_code_name()` 返回的股票代码全部是 6 位数
   - 共 5497 只，全部是标准格式

3. **潜在风险**
   - 如果 stock_list 表中混入了短代码，批量更新时会获取这些短代码的数据并保存
   - 虽然查询时已经过滤，但在保存时没有验证

#### 2. save_stock_list 函数添加验证
```python
def save_stock_list(self, stocks: pd.DataFrame) -> int:
    # ... 原有逻辑
    rows = []
    skipped = 0
    for _, row in stocks.iterrows():
        code = str(row['code'])
        if len(code) != 6:
            skipped += 1
            continue
        rows.append((code, row['name'], 'A股', date.today().isoformat()))

    if skipped > 0:
        print(f"跳过 {skipped} 个无效股票代码（长度不为6）")

    if not rows:
        return 0
    # ... 原有逻辑
```
